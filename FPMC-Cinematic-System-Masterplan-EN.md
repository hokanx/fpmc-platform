## 2. Anatomy of the portfolio sites — the replication blueprint

Every niche site in the Doc Mo portfolio follows the same construction. This is the structure FPMC replicates 1:1 (it matches the playbook — the categories map directly to our niche templates):

### 2.1 The universal site architecture (all niches)

```
[1] HERO — scroll-scrubbed video (canvas frame sequence):
    Scrolling drives the film. Brand name in massive type + one-line motto.
[2] STORY/PHILOSOPHY — pinned text reveals over clip 2,
    one sentence per scroll step.
[3] OFFER — menu / services / grid / programs
    (niche-specific, real prices).
[4] PROOF — counters, gallery, before/after, testimonials.
[5] CONVERSION — form / booking / reservation + contact,
    hours, map.
```

**Tech DNA of every site:** Lenis smooth scroll · canvas scrub of the hero video (frames, not `<video currentTime>`) · pinned sections · scroll-synced counters · ONE accent color · film grain · sparse, confident copy · mobile pass built in · videos web-compressed (~90% smaller).

## 7. Tech & pipeline (binding for all builds)

- **Generation:** Higgsfield MCP / Seedance 2.0. Hero image always first, passed as reference into every clip. Journeys: chained final-frame → start-frame.
- **Build:** single-file-friendly static sites · Lenis smooth scroll · canvas frame scrub · pinned reveals · scroll-synced counters · one accent · grain · sparse copy.
- **Verify:** localhost, full scroll in both directions, seams, hovers, forms, mobile. "Done" = verified.
- **Deliver:** compress videos (~90%) → GitHub → Cloudflare Pages → custom domain.
- **Client mode later:** the client's real photos as Seedance references (their food, their shop, their car), real menu/prices/hours. Critical elements (faces) filmed for real, AI only modifies — consistent with your risk chapter 9.1.
